# 产品需求文档：All about 个人主页 - V1.1

## 1. 综述 (Overview)

### 1.1 项目背景与核心问题
在 V1.0 建立了基本的个人展示与互动功能后，V1.1 版本旨在扩展“工具箱”属性。用户希望在主页直接提供一些实用的微型工具（如图片压缩、AI 工具快捷入口），将个人主页从单纯的信息展示升级为“实用工具站”，增加用户的回访率和页面的实用价值。

### 1.2 核心业务流程 / 用户旅程地图
1.  **阶段一：初次访问与印象建立** - 访客进入主页，通过 Identity Card（身份卡片）建立第一印象。
2.  **阶段二：内容浏览与探索** - 访客浏览作品集、GitHub 统计、天气等卡片。
3.  **阶段三：工具使用 (New)** - 访客在页面底部发现“常用工具栏”，使用图片压缩功能或跳转至 AI 工具。
4.  **阶段四：互动与反馈** - 访客在留言板互动。

## 2. 用户故事详述 (User Stories)

### 阶段三：工具使用 (New)

---

#### **US-04: 常用工具栏 (Tools Dashboard Card)**
*   **价值陈述 (Value Statement)**:
    *   **作为** 访客/博主本人
    *   **我希望** 在主页底部能直接使用一些高频小工具（如图片压缩、AI 入口）
    *   **以便于** 提高工作效率，无需打开额外的书签或应用
*   **业务规则与逻辑 (Business Logic)**:
    1.  **布局位置**: 位于 Bento Grid 底部，通栏布局 (col-span-4)，确保视觉上的稳重感和工具的易达性。
    2.  **功能模块**:
        *   **图片压缩 (Image Compressor)**: 提供一个简单的拖拽上传区域，前端实现图片压缩（使用 browser-image-compression 等库），支持下载。
        *   **AI 工具箱 (AI Hub)**: 展示几个常用 AI 工具的快捷图标/入口（如 ChatGPT, Claude, Midjourney 等），或者是内置的一个简单 AI 对话框（视 API key 情况而定，V1.1 建议先做快捷导航或简单的 Prompt 生成器）。
    3.  **交互设计**: 
        *   每个工具以“子卡片”或“图标”形式排列。
        *   点击“图片压缩”弹出一个模态框 (Modal) 或在卡片内展开操作区。
*   **验收标准 (Acceptance Criteria)**:
    *   **场景1: 图片压缩**
        *   **GIVEN** 用户点击工具栏的“图片压缩”图标
        *   **WHEN** 拖入一张 5MB 的图片
        *   **THEN** 显示压缩进度，完成后显示“下载”按钮，且文件大小显著减小。
    *   **场景2: AI 工具导航**
        *   **GIVEN** 用户查看 AI 工具区
        *   **WHEN** 点击某个 AI 图标
        *   **THEN** 新标签页打开对应的 AI 工具网站。
